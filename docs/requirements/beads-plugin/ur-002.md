## UR-002: View Ready Issues

**Type:** User Requirement
**Status:** Finalized

### Description
Users can view issues ready to work (no blockers) using `:BdReady` command, with optional filtering by assignee.

### Rationale
A common workflow is to find issues that are actionable immediately without waiting on dependencies.

### Source
Spec Section 1 (Telescope Pickers - Ready Issues)

### Acceptance Criteria
- [ ] `:BdReady` command opens Telescope picker
- [ ] Only displays issues with no blockers (from `bd ready`)
- [ ] Same picker interface as `:BdList`
- [ ] `:BdReady` supports optional `--assigned <user>` filter
- [ ] Selection and preview actions work same as `:BdList`
