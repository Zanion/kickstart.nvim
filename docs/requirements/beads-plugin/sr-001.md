## SR-001: bd CLI Invocation Layer

**Type:** System Requirement
**Status:** Draft

### Description
The plugin must provide a robust Lua wrapper for invoking the `bd` CLI with JSON output support.

### Priority
- [ ] Critical

### Dependencies
- None

### Constraints
- Must handle `bd --json` output parsing
- Must support all Phase 1 commands: list, ready, blocked, search, query, show, status
- Must handle missing bd binary gracefully with helpful error message
- Must support workspace-relative database path (via `BD_PATH` or cwd-based discovery)

### Acceptance Criteria
- [ ] `beads.cli.run("list", opts)` executes `bd list --json` and parses output
- [ ] `beads.cli.run("ready", opts)` executes `bd ready --json`
- [ ] `beads.cli.run("blocked", opts)` executes `bd blocked --json`
- [ ] `beads.cli.run("search", query)` executes `bd search <query> --json`
- [ ] `beads.cli.run("query", expr)` executes `bd query <expr> --json`
- [ ] `beads.cli.show(id)` executes `bd show <id> --json`
- [ ] `beads.cli.show_url(id)` executes `bd show <id> --url` and returns URL
- [ ] `beads.cli.status()` executes `bd status --json`
- [ ] `beads.cli.graph(id, opts)` executes `bd graph [id] --json`
- [ ] All CLI functions return structured Lua tables, not raw JSON strings
- [ ] Error handling: clear error messages when bd binary not found or command fails
- [ ] Async job execution via nvim `jobstart` or `uv.spawn` for non-blocking UI
