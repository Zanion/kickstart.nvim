## SR-010: Plugin Configuration API

**Type:** System Requirement
**Status:** Draft

### Description
The plugin must provide a `setup()` function for user configuration with sensible defaults.

### Priority
- [ ] Critical

### Dependencies
- None

### Constraints
- Must use lazy configuration pattern (setup before first use)
- Must validate configuration values
- Must use functional merge for nested options

### Acceptance Criteria
- [ ] `require("beads").setup(opts)` accepts configuration table
- [ ] Supported options: binary (string), list_opts (table), ready_opts, blocked_opts, keymaps (table)
- [ ] Default binary: "bd"
- [ ] Default keymaps: open = "<CR>", show = "<leader>i", close = "q"
- [ ] Validation: binary must be string, keymaps must be table
- [ ] Error handling: clear message if required option is invalid type
