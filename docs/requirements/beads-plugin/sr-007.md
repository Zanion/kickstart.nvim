## SR-007: Neovim Command Registration

**Type:** System Requirement
**Status:** Draft

### Description
The plugin must register all user-facing commands (:BdList, :BdReady, etc.) with proper argument handling.

### Priority
- [ ] Critical

### Dependencies
- SR-001, SR-002, SR-005, SR-006

### Constraints
- Must use nvim `vim.api.nvim_create_user_command` for registration
- Must support both arguments and ranges where applicable

### Acceptance Criteria
- [ ] `:BdList` - Opens list picker
- [ ] `:BdList --assigned <user>` - Opens list picker with assignee filter
- [ ] `:BdReady` - Opens ready picker
- [ ] `:BdReady --assigned <user>` - Opens ready picker with assignee filter
- [ ] `:BdBlocked` - Opens blocked picker
- [ ] `:BdBlocked --assigned <user>` - Opens blocked picker with assignee filter
- [ ] `:BdSearch` - Opens search picker
- [ ] `:BdQuery` - Opens query picker  
- [ ] `:BdShow <id>` - Opens issue detail in vertical split
- [ ] `:BdShowH <id>` - Opens issue detail in horizontal split
- [ ] `:BdOpen <id>` - Opens issue in browser
- [ ] `:BdStatus` - Opens database status in vertical split
- [ ] `:BdGraph` - Opens dependency graph in float
- [ ] `:BdGraph <id>` - Opens graph centered on issue
- [ ] `:BdGraph --all` - Opens graph showing all issues
- [ ] All commands handle missing arguments with helpful error messages
