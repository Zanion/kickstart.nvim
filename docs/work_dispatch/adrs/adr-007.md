# ADR-007: Merge Workflow Implementation

## Status
Proposed

## Context
Merge should create a PR with proper linking to the bead. We need a consistent, automated workflow.

## Decision
Merge action performs:
1. Change to worktree directory
2. `git add -A && git commit -m "feat: {bead title}"` (if uncommitted changes)
3. `git push -u origin {branch}`
4. `gh pr create --title "feat({bead-id}): {title}" --body "Implements {bead-id}\n\nCloses #{bead-number}"`
5. `bd close {bead_id} --reason "Implemented"`
6. Show notification with PR URL

## Workflow Steps

### 1. Validate State

```lua
local function merge.validate(worktree_id)
  local worktree = registry.get(worktree_id)
  
  -- Check worktree exists
  if not worktree or not vim.fn.isdirectory(worktree.path) then
    return false, "Worktree not found"
  end
  
  -- Check status allows merge
  if worktree.status == "done" then
    return false, "Already merged"
  end
  
  if worktree.status == "rejected" then
    return false, "Work was rejected"
  end
  
  -- Check remote configured
  local remote = git.get_remote()
  if not remote then
    return false, "No git remote configured"
  end
  
  return true
end
```

### 2. Commit Changes

```lua
local function merge.commit(worktree)
  -- Check for changes
  local has_changes = git.has_changes(worktree.path)
  if not has_changes then
    return true  -- Nothing to commit
  end
  
  -- Stage all changes
  git.add_all(worktree.path)
  
  -- Generate commit message
  local msg = string.format([[
feat(%s): %s

%s

Closes #%s
]],
    worktree.bead_id,
    worktree.bead_title,
    worktree.bead_title,
    extract_bead_number(worktree.bead_id)
  )
  
  -- Commit
  return git.commit(worktree.path, msg)
end
```

### 3. Push Branch

```lua
local function merge.push(worktree)
  local result = vim.fn.system({
    "git", "push",
    "-u", "origin", worktree.branch,
  }, nil, { cwd = worktree.path })
  
  if vim.v.shell_error ~= 0 then
    -- Handle push failure
    if result:match("rejected") then
      return false, "Push rejected - branch may need rebasing"
    elseif result:match("no upstream") then
      return false, "Remote branch not found"
    else
      return false, "Push failed: " .. result
    end
  end
  
  -- Extract commit hash
  local commit = result:match("(%x+)%.%.%s*(%x+)")
  return true, commit
end
```

### 4. Create PR

```lua
local function merge.create_pr(worktree, commit)
  -- Build PR title and body
  local title = string.format("feat(%s): %s",
    worktree.bead_id,
    worktree.bead_title
  )
  
  local body = string.format([[
## Summary
<!-- Brief description of changes -->

## Changes
<!-- Summary of what was implemented -->

## Testing
- [ ] Tests added/updated
- [ ] Manual testing completed

## Notes
<!-- Any additional context -->

Implements **%s**

Closes #%s
]],
    worktree.bead_id,
    extract_bead_number(worktree.bead_id)
  )
  
  -- Use gh CLI
  local cmd = {
    "gh", "pr", "create",
    "--title", title,
    "--body", body,
    "--head", worktree.branch,
    "--base", "master",  -- or main
  }
  
  local result = vim.fn.system(cmd, nil, { cwd = worktree.path })
  
  if vim.v.shell_error ~= 0 then
    -- Check if PR already exists
    if result:match("pull request already exists") then
      -- Find existing PR URL
      return merge.find_existing_pr(worktree)
    end
    return false, "PR creation failed: " .. result
  end
  
  -- Extract PR URL
  local url = result:match("(https://github%.com/[^\n]+)")
  return true, url
end
```

### 5. Update Bead

```lua
local function merge.close_bead(worktree)
  local result = vim.fn.system({
    "bd", "close", worktree.bead_id,
    "--reason", "Implemented",
  })
  
  if vim.v.shell_error ~= 0 then
    -- Log but don't fail
    vim.notify("Warning: Failed to close bead: " .. result, "warn")
    return false
  end
  
  return true
end
```

### 6. Update Registry

```lua
local function merge.complete(worktree_id, pr_url, commit)
  registry.update(worktree_id, {
    status = "done",
    pr_url = pr_url,
    merge_commit = commit,
    merged_at = os.date("!%Y-%m-%dT%H:%M:%SZ"),
  })
end
```

## Consequences

### Easier
- Standardized commit/PR format
- Full trace from bead to PR
- User can review before/after push
- Automated bead closing

### Harder
- Requires gh CLI
- Branch must be pushed successfully
- Conflict detection if main has advanced
- PR template customization

## Error Handling

| Error | Recovery |
|-------|----------|
| Uncommitted changes | Auto-commit or abort |
| Push rejected | Offer rebase or abort |
| PR exists | Return existing URL |
| gh not installed | Show setup instructions |
| Bead close fails | Log warning, continue |
| Network error | Retry or manual option |
