# ADR-002: IPC Notification via NVIM_LISTEN_ADDRESS

## Status
Proposed

## Context
Agents need to signal when they need user input. The existing agent.lua already uses `NVIM_LISTEN_ADDRESS` via `NVIM=vim.v.servername` for basic communication.

## Decision
Agents send structured notifications via msgpack-rpc to the Neovim socket. The plugin registers a handler that:
1. Receives the notification
2. Updates the agent registry
3. Triggers a notify via nvim-notify
4. Updates the picker if open

## Alternatives Considered

### Option A: Marker file in worktree
- Agent writes `.needs_input` file
- Plugin polls for changes
- Pros: Simple to implement
- Cons: Polling overhead, latency, file pollution

### Option B: IPC via NVIM_LISTEN_ADDRESS (CHOSEN)
- Agent connects via msgpack-rpc
- Sends structured notifications
- Pros:
  - Real-time communication
  - Rich structured data
  - Reuses existing pattern
- Cons:
  - Requires msgpack-rpc implementation
  - Socket lifecycle management

### Option C: Special log pattern
- Agent outputs `[NEEDS_INPUT]` to stdout
- Plugin tails terminal output
- Pros: No socket needed
- Cons: Fragile, requires output parsing

## Implementation

### Neovim Side
```lua
-- Register RPC handler
vim.rpcstart(vim.v.servername, {
  work_dispatch_notify = function(event, payload)
    if event == "needs_input" then
      handle_needs_input(payload)
    elseif event == "status" then
      handle_status(payload)
    end
  end,
})

local function handle_needs_input(payload)
  -- Update registry
  registry.update(payload.worktree_id, {
    needs_input = true,
    needs_input_since = os.date("!%Y-%m-%dT%H:%M:%SZ"),
  })
  
  -- Show notification
  require("notify")(
    payload.message,
    "info",
    { title = "Agent Input Required" }
  )
  
  -- Refresh picker
  picker.refresh()
end
```

### Agent Side (Conceptual)
```lua
-- Agent pseudo-code for notification
local function notify_needs_input(message)
  local socket = os.getenv("NVIM_LISTEN_ADDRESS")
  if not socket then return end
  
  local rpc = msgpack_rpc.connect(socket)
  rpc:notify("work_dispatch_notify", "needs_input", {
    worktree_id = os.getenv("WORKTREE_ID"),
    message = message,
    timestamp = os.date("!%Y-%m-%dT%H:%M:%SZ"),
  })
end
```

## Notification Schema

```lua
-- Needs Input
{
  type = "needs_input",
  worktree_id = "uuid",
  message = "Should I proceed with the migration?",
  options = { "Yes", "No", "Ask Later" },
}

-- Status Update
{
  type = "status",
  worktree_id = "uuid",
  level = "info",  -- info, warn, error
  message = "Running tests...",
}

-- Complete
{
  type = "complete",
  worktree_id = "uuid",
  summary = "Implementation finished",
  files_changed = 5,
}
```

## Consequences

### Easier
- Reuses existing IPC pattern from agent.lua
- Real-time communication without polling
- Agents can send rich notifications with metadata

### Harder
- Requires msgpack-rpc implementation or helper library
- Socket lifecycle management
- Error handling when Neovim isn't listening
- Agent must support socket-based notifications
