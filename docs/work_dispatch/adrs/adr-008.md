# ADR-008: Worktree Naming Convention

## Status
Proposed

## Context
Worktrees need consistent, predictable names that indicate their purpose and support parallel agents.

## Decision
Naming pattern: `feature-{bead-id}-{slug}` with incrementer:
- Single agent: `feature-bd-42-add-login`
- Multiple agents: `feature-bd-42-1-add-login`, `feature-bd-42-2-add-login`

Slug is derived from bead title:
- Lowercase
- Replace spaces with hyphens
- Remove special characters
- Truncate to 50 chars

## Name Components

```
feature-{bead-id}[-{counter}]-{slug}

feature    - Fixed prefix indicating feature worktree
{bead-id}  - Beads issue ID (e.g., bd-42, nvim-abc123)
{counter}  - Optional, for parallel agents (1, 2, 3...)
{slug}     - Sanitized bead title
```

## Slug Generation

```lua
local function generate_slug(title)
  -- Truncate before sanitization if very long
  title = title:sub(1, 100)
  
  -- Lowercase
  title = title:lower()
  
  -- Replace spaces and underscores with hyphens
  title = title:gsub("[%s_]+", "-")
  
  -- Remove special characters (keep alphanumeric and hyphens)
  title = title:gsub("[^a-z0-9%-]", "")
  
  -- Collapse multiple hyphens
  title = title:gsub("%-+", "-")
  
  -- Remove leading/trailing hyphens
  title = title:gsub("^%-", ""):gsub("%-$", "")
  
  -- Truncate to max length
  title = title:sub(1, 50)
  
  return title
end
```

### Examples

| Bead Title | Generated Slug |
|------------|----------------|
| "Add login functionality" | `add-login-functionality` |
| "Fix bug #123 in auth" | `fix-bug-123-in-auth` |
| "API v2!!! 🚀" | `api-v2` |
| "What's new?" | `whats-new` |

## Counter Logic

```lua
local function get_next_counter(bead_id, existing_names)
  local max_counter = 0
  
  for _, name in ipairs(existing_names) do
    -- Match pattern: feature-{bead-id}[-{counter}]-{slug}
    local counter = name:match(
      string.format("feature%-{0}%-(%d+)%%-", bead_id)
    )
    if counter then
      max_counter = math.max(max_counter, tonumber(counter))
    end
  end
  
  return max_counter + 1
end

local function build_worktree_name(bead_id, slug, counter)
  if counter == 1 then
    return string.format("feature-%s-%s", bead_id, slug)
  else
    return string.format("feature-%s-%d-%s", bead_id, counter, slug)
  end
end
```

### Counter Examples

| Bead | Counter | Worktree Name |
|------|---------|---------------|
| bd-42 | 1 | `feature-bd-42-add-login` |
| bd-42 | 2 | `feature-bd-42-1-add-login` |
| bd-42 | 3 | `feature-bd-42-2-add-login` |

## Full Name Generation

```lua
local function generate_worktree_name(bead_id, bead_title)
  local slug = generate_slug(bead_title)
  local existing = registry.get_names_by_bead(bead_id)
  local counter = get_next_counter(bead_id, existing)
  
  return build_worktree_name(bead_id, slug, counter)
end
```

## Alternatives Considered

### Option A: UUID-based names
- e.g., `worktree-abc123`
- Pros: Guaranteed unique
- Cons: Hard to identify bead from name

### Option B: Timestamp-based
- e.g., `feature-2024-01-15-10-30-add-login`
- Pros: Chronological ordering
- Cons: Long names, no counter support

### Option C: User-defined
- User picks name on dispatch
- Pros: Full flexibility
- Cons: Inconsistent, requires validation

## Consequences

### Easier
- Human-readable names
- Easy to identify bead from worktree name
- Collision prevention with counter
- Consistent pattern across all worktrees

### Harder
- Slug generation may produce duplicates (handled by counter)
- Maximum path length on some systems (255 chars)
- Slug sanitization edge cases

## Branch Naming

Branch names mirror worktree names:

```lua
local function build_branch_name(worktree_name)
  -- Branch name = worktree name (already valid)
  return worktree_name
end

-- Git worktree add command
git worktree add -b feature-bd-42-add-login \
  .worktree/feature-bd-42-add-login \
  HEAD
```

## Validation

```lua
local function validate_worktree_name(name)
  -- Check length
  if #name > 255 then
    return false, "Name too long (max 255 chars)"
  end
  
  -- Check valid characters
  if not name:match("^[a-z0-9%-]+$") then
    return false, "Name contains invalid characters"
  end
  
  -- Check reserved names
  local reserved = { "master", "main", "develop", "HEAD" }
  for _, r in ipairs(reserved) do
    if name == r then
      return false, "Reserved branch name"
    end
  end
  
  return true
end
```
