# UR-001: Worktree Isolation per Feature

## Status
Draft

## Description
Agents work in isolated git worktrees stored under `.worktree/` (gitignored), where each feature implementation gets its own worktree and branch.

## Rationale
Isolation prevents agents from conflicting with each other's work and keeps the main branch clean until work is reviewed and merged.

## Source
Feature requirement from AGENT_WORKTREES_MANAGER.md

## Acceptance Criteria

- [ ] Worktrees are created under `.worktree/` directory
- [ ] Each worktree has a dedicated branch for the feature
- [ ] `.worktree/` directory is gitignored
- [ ] Worktrees persist indefinitely for inspection (never auto-deleted)
- [ ] User can configure the worktree root location
