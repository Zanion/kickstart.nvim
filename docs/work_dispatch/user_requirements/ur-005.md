# UR-005: Agent Picker with Preview

## Status
Draft

## Description
User can open a picker to see all open agents conducting work in a floating telescope pane, with live preview of terminal output.

## Rationale
Centralized visibility into all active agents allows the user to monitor progress and quickly access specific agents.

## Source
Feature requirement from AGENT_WORKTREES_MANAGER.md

## Picker UI Layout

```
┌─────────────────────────────────────────────────────────────┐
│                    Active Agents                             │
├─────────────────────────────────────────────────────────────┤
│  │ ● gemini │ bd-42 │ feature-42-add-login │ ⚠️ needs input │
│  │ ○ gemini │ bd-41 │ feature-41-fix-bug    │ running       │
│  │ ○ claude │ bd-40 │ feature-40-refactor   │ done          │
├─────────────────────────────────────────────────────────────┤
│  [Preview Pane - Live Terminal Output]                       │
│                                                              │
│  > Implementing feature...                                   │
│  > Running tests...                                          │
│  > Should I proceed with the migration?                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘

Legend: ● = selected, ⚠️ = awaiting input
```

## Display Fields
| Field | Description |
|-------|-------------|
| Agent Icon | Visual identifier for agent type (gemini/cursor/claude/opencode) |
| Bead ID | Beads issue ID (e.g., bd-42) |
| Branch Name | Git branch for the worktree |
| Status | Current agent status with icon |

## Status Icons
- `●` Running (green) - Agent actively working
- `⚠️` Needs Input (yellow) - Agent awaiting user response
- `✓` Done (blue) - Work complete, ready for review
- `✗` Rejected (red) - Work rejected, kept for inspection

## Acceptance Criteria

- [ ] Single keybind opens the agent picker
- [ ] Picker shows all active agent worktrees
- [ ] Preview pane shows live terminal output (scrollback)
- [ ] Visual indication distinguishes agents awaiting input
- [ ] Preview updates in real-time as agent produces output

## Mapped System Requirements
- SR-010: Agent Picker UI
- SR-011: Terminal Preview Component
- SR-012: Agent Focus Action

## Keybindings (within picker)
| Key | Action |
|-----|--------|
| `<CR>` | Focus agent terminal |
| `<leader>m` | Merge agent work |
| `<leader>r` | Reject agent work |
| `<leader>R` | Reset agent |
| `<leader>d` | Delete worktree (with confirmation) |
| `q` / `<Esc>` | Close picker |

## Sorting Options
- By status (needs_input first)
- By bead priority
- By creation time (newest/oldest)
- By agent type
