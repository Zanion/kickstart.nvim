# UR-006: Merge and Reject Actions

## Status
Draft

## Description
User can merge or reject agent work via keybinds from the picker. Merge creates a PR; reject keeps worktree for inspection and resets the agent.

## Rationale
Direct action from the picker enables efficient workflow without switching contexts.

## Source
Feature requirement from AGENT_WORKTREES_MANAGER.md

## Merge Action

### User Flow
1. User selects agent in picker
2. User presses `<leader>m`
3. System:
   - Commits any uncommitted changes
   - Pushes branch to remote
   - Creates PR with proper format
   - Closes bead as "Implemented"
   - Shows success notification with PR URL
4. Worktree remains for post-merge inspection

### PR Format
```markdown
## Summary
- {bullet point of changes}

## Testing
- {how changes were tested}

## Notes
- {any additional context}

Closes #{bead-number}
```

### Acceptance Criteria
- [ ] `<leader>m` triggers merge action (creates PR)
- [ ] Branch is pushed to remote before PR creation
- [ ] PR title includes bead ID and feature name
- [ ] PR body references the bead
- [ ] Bead is closed with "Implemented" reason
- [ ] Success notification shows PR URL

## Reject Action

### User Flow
1. User selects agent in picker
2. User presses `<leader>r`
3. System:
   - Prompts for rejection reason (optional)
   - Updates bead status to "rejected" (not closed)
   - Marks agent session as rejected
   - Keeps worktree and branch intact
   - Optionally kills agent terminal
4. User can inspect work and optionally reset agent later

### Important Notes
- Reject does NOT close the bead
- Bead can be reset and picked up again
- Worktree persists for inspection

### Acceptance Criteria
- [ ] `<leader>r` triggers reject action (keeps worktree, resets bead)
- [ ] Bead status updated to "rejected"
- [ ] Worktree and branch remain intact
- [ ] Agent terminal closed
- [ ] Confirmation notification shown

---

## Reset Action

### User Flow
1. User selects rejected agent in picker
2. User presses `<leader>R`
3. System:
   - Confirms reset intent
   - Kills existing agent terminal
   - Resets bead status to "ready"
   - Re-spawns agent in same worktree
   - Agent re-reads BEADS_CONTEXT.md

### Acceptance Criteria
- [ ] `<leader>R` triggers reset action (relaunches agent on same worktree)
- [ ] Bead status reset to previous state
- [ ] Agent terminal killed and respawned
- [ ] needs_input status cleared
- [ ] Session timestamp updated

## Edge Cases
- **Uncommitted changes on merge**: Auto-commit with standardized message
- **Remote ahead of local on merge**: Pull and resolve or abort
- **Agent still running on reject**: Kill terminal before marking rejected
- **Bead already closed**: Prevent merge/reject, show warning
