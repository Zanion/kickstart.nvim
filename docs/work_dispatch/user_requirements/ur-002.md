# UR-002: Beads-Aware Work Assignment

## Status
Draft

## Description
Agents pick up work from beads tasks that have been fully scoped by architect/planner. The user manually dispatches beads to selected agents.

## Rationale
Beads provides structured task tracking with dependencies. Only architect/planner-scoped tasks should be worked on to ensure proper requirements exist.

## Source
Feature requirement from AGENT_WORKTREES_MANAGER.md

## User Flow
1. User invokes picker showing available beads from `bd ready`
2. User selects a specific bead to dispatch
3. User selects a specific agent to handle the bead
4. System creates worktree, updates bead state, spawns agent

## Acceptance Criteria

- [ ] User can invoke a picker showing available beads from `bd ready`
- [ ] User can select a specific agent to dispatch the bead to
- [ ] Dispatched bead is updated to "in_progress" state with agent assignment
- [ ] Agent receives the bead context/instructions in its worktree

## Mapped System Requirements
- SR-004: Beads Integration Layer
- SR-005: Dispatch Command
- SR-006: Bead Context Injection

## Configuration
```lua
require("work_dispatch").setup({
  -- Worktree configuration
  worktree_root = ".worktree/",  -- or "~/.agent-worktrees"
})
```

## Edge Cases
- **Bead already assigned**: Show warning, allow reassignment
- **Agent not available**: Disable dispatch, show status
- **Worktree already exists for bead**: Offer to resume or create parallel
