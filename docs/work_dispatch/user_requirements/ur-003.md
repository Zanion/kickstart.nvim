# UR-003: Agent Workflow Compliance

## Status
Draft

## Description
Agents follow the beads-workflow for implementation, including landing procedures (commit, push, PR creation).

## Rationale
Consistency with the established beads workflow ensures proper version control practices are followed.

## Source
AGENTS.md and AGENT_WORKTREES_MANAGER.md

## Beads Workflow Requirements

### Agent Instructions
When dispatched to a worktree, agents must:
1. Read `BEADS_CONTEXT.md` for task instructions
2. Follow the beads-skill workflow (Claim → Analyze → Test → Implement → Close)
3. Commit with standardized messages
4. Push and create PR when landing

### Workspace Setup
The worktree should contain:
- `BEADS_CONTEXT.md` - Full bead details and acceptance criteria
- `AGENTS.md` - Agent instructions (from main repo or custom)
- `.beads/` - Local beads data if needed

## Acceptance Criteria

- [ ] Agent workspace contains beads workflow instructions
- [ ] Agent commits follow standardized commit messages
- [ ] Agent pushes work to remote when landing
- [ ] Agent creates PRs via the established workflow

## Mapped System Requirements
- SR-006: Bead Context Injection
- SR-007: Agent Session Management

## Standardized Commit Format
```
feat({bead-id}): {short description}

{Bead title}

Closes #{bead-number}
```
