# UR-007: Multi-Agent Parallel Implementation

## Status
Draft

## Description
Multiple agents can work on the same bead in parallel, each on their own worktree, enabling competing implementations.

## Rationale
Parallel implementations allow for comparison and potentially faster delivery of features. The user can evaluate different approaches and choose the best one.

## Source
User clarification on multi-agent coordination

## Use Cases

### Competitive Implementation
- Dispatch same bead to multiple agents (gemini, claude, opencode)
- Each agent produces its own solution
- User reviews all implementations
- Merges best one, rejects others

### Divide and Conquer
- Architect splits bead into sub-tasks
- Different agents work on different aspects
- Results combined or compared

### Parallel Testing
- Same feature implemented differently
- Both tested simultaneously
- Better solution merged

## Worktree Naming

When multiple agents work on same bead:

| Instance | Worktree Name | Branch Name |
|----------|--------------|-------------|
| 1st | `feature-42-add-login` | `feature-bd-42-add-login` |
| 2nd | `feature-42-1-add-login` | `feature-bd-42-1-add-login` |
| 3rd | `feature-42-2-add-login` | `feature-bd-42-2-add-login` |

### Naming Algorithm
```
if first_agent_on_bead:
    name = "feature-{bead-id}-{slug}"
    branch = "feature-{bead-id}-{slug}"
else:
    # Find highest existing counter
    counter = get_max_counter(bead_id) + 1
    name = "feature-{bead-id}-{counter}-{slug}"
    branch = "feature-{bead-id}-{counter}-{slug}"
```

## Independent Merge/Reject

Each parallel worktree is independent:
- Can merge one implementation
- Can reject others
- Each has own bead reference
- Bead closes when first merge completes

## Acceptance Criteria

- [ ] Same bead can be dispatched to multiple agents
- [ ] Each agent gets a uniquely named worktree (e.g., `feature-bd-42-1`, `feature-bd-42-2`)
- [ ] Picker shows all parallel implementations
- [ ] User can merge one and reject others
- [ ] Counter increments correctly for same bead

## Configuration

```lua
require("work_dispatch").setup({
  parallel = {
    max_agents_per_bead = 5,  -- Limit parallel implementations
    auto_increment = true,   -- Auto-add counter to names
  }
})
```

## Edge Cases
- **Max parallel reached**: Show warning, prevent dispatch
- **Counter collision**: Handle gracefully, find next available
- **Same agent type twice**: Allow, differentiate by counter
