# UR-004: Agent Notification for User Input

## Status
Draft

## Description
Agents can send notifications to the Neovim user via IPC to signal when they need user input.

## Rationale
Agents operate autonomously but must be able to request human input when decisions or approvals are needed.

## Source
Feature requirement from AGENT_WORKTREES_MANAGER.md

## User Flow
1. Agent encounters situation requiring input (e.g., ambiguous decision, security concern)
2. Agent sends notification via IPC to Neovim
3. Notification appears via nvim-notify
4. Picker updates to show visual indicator
5. User responds by focusing the agent terminal

## Notification Types

### Input Request
```json
{
  "type": "needs_input",
  "bead_id": "bd-42",
  "agent_id": "uuid",
  "message": "Should I proceed with the migration or create a new schema?",
  "timestamp": "ISO8601"
}
```

### Status Update
```json
{
  "type": "status",
  "bead_id": "bd-42",
  "agent_id": "uuid",
  "status": "analyzing|implementing|testing",
  "message": "Running tests...",
  "timestamp": "ISO8601"
}
```

## Acceptance Criteria

- [ ] Agent can send notifications to the NVIM_LISTEN_ADDRESS
- [ ] Notifications appear via nvim-notify
- [ ] Visual indication in the picker shows agents awaiting input
- [ ] User can respond to notification by opening the relevant agent

## Mapped System Requirements
- SR-008: IPC Notification System
- SR-009: Needs-Input Detection and Display

## IPC Protocol
Agents communicate via msgpack-rpc to the Neovim socket:
```lua
-- Neovim side handler
vim.fn.rpcconnect(socket_path, 'work_dispatch', {
  notify = function(method, args)
    if method == 'needs_input' then
      -- Handle needs_input notification
    end
  end
})
```
