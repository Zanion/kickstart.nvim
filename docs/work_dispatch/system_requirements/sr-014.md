# SR-014: Reject Action

## Status
Draft

## Priority
High

## Dependencies
- SR-003: Worktree Registry
- SR-004: Beads Integration Layer

## Description
Reject agent work, keeping worktree for inspection and resetting bead.

## Constraints

### Worktree Preservation
- Worktree is NEVER deleted on reject
- Branch and files remain intact
- Agent terminal is closed

### Bead State
- Bead is NOT closed
- Status set to "rejected"
- Can be reset and retried

## Reject Process

```lua
local function reject(worktree_id, reason)
  local worktree = registry.get(worktree_id)
  if not worktree then
    return error("Worktree not found")
  end
  
  -- 1. Prompt for reason (optional)
  if reason == nil then
    reason = vim.fn.input("Rejection reason (optional): ")
  end
  
  -- 2. Kill agent terminal if running
  if worktree.terminal_id then
    terminal.kill(worktree.terminal_id)
  end
  
  -- 3. Update registry
  registry.update(worktree_id, {
    status = "rejected",
    rejected_at = os.date("!%Y-%m-%dT%H:%M:%SZ"),
    rejection_reason = reason ~= "" and reason or nil,
  })
  
  -- 4. Update bead status (not closed)
  beads.set_status(worktree.bead_id, "rejected")
  
  -- 5. Notify
  require("notify")(
    string.format("Agent work rejected for %s", worktree.bead_id),
    "warn",
    {
      title = "Agent Rejected",
      timeout = 3000,
    }
  )
  
  return { success = true }
end
```

## Visual Feedback

Picker updates to show:
- Status icon: `✗` (red)
- Row highlight: red tint
- Sorted to end of list

## Bead State Transition

```
Current: in_progress, running
     ↓
After reject: rejected
     ↓
After reset: ready (or in_progress if re-dispatched)
```

## Functions

### `actions.reject(entry)`
Main reject action.

**Parameters:**
- `entry` (table, required) - Picker entry
- `reason` (string, optional)

**Returns:** boolean success

### `reject.confirm(worktree_id)`
Shows confirmation prompt before reject.

```lua
function reject.confirm(worktree_id)
  local worktree = registry.get(worktree_id)
  
  local confirmed = vim.fn.confirm(
    string.format(
      "Reject agent work for %s?\n\n".
      "Worktree will be kept for inspection.\n".
      "Bead will be marked as rejected.",
      worktree.bead_id
    ),
    { "Reject", "Cancel" },
    1,
    "Warning"
  )
  
  if confirmed == 1 then
    return reject(worktree_id)
  else
    return { success = false, cancelled = true }
  end
end
```

## Inspection Access

After reject, user can:
1. Open picker and select rejected worktree
2. Press `<CR>` to focus terminal
3. Review changes with `git diff`
4. Manually commit/push if desired
5. Reset agent to retry

## Reset After Reject

See SR-015 for reset action.

## State Machine

```
[ready] ────────────────────────────────┐
    │ dispatch                            │
    ▼                                     │
[in_progress] ──── agent running ──→ [running]
    │                                     │
    │                                     ▼
    │                              [needs_input]
    │                                     │
    │                                     ▼
    └───────────── agent done ←──── [running]
                            │ merge
                            ▼
                        [done]
                            
[running] ── reject ─→ [rejected]
                            │
                            ▼ reset
                        [ready]
```

## Acceptance Criteria

- [ ] `<leader>r` triggers reject action (keeps worktree, resets bead)
- [ ] Bead status updated to "rejected"
- [ ] Worktree and branch remain intact
- [ ] Agent terminal closed
- [ ] Confirmation notification shown

## Edge Cases

| Case | Handling |
|------|----------|
| Agent still running | Kill terminal before marking |
| Uncommitted changes | Keep, don't auto-commit |
| Multiple parallel agents | Only reject selected one |
| Bead already done | Prevent reject, show warning |
