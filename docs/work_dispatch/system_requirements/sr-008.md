# SR-008: IPC Notification System

## Status
Draft

## Priority
Critical

## Description
Mechanism for agents to send notifications to Neovim via NVIM_LISTEN_ADDRESS.

## Dependencies
None (uses Neovim's built-in RPC)

## Constraints

### Protocol
- Use Neovim's msgpack-rpc protocol
- Connect via Unix socket at NVIM_LISTEN_ADDRESS
- Send `plugin_notify` method for notifications

### Socket Path
- Neovim sets `NVIM_LISTEN_ADDRESS` env var
- Format: `/tmp/nvim-12345` or similar
- Agents receive via `NVIM_LISTEN_ADDRESS` env var

## IPC Protocol

### Message Format (msgpack-rpc)
```
[msg_type, msg_id, method, params]

msg_type: 0 (request) or 2 (notification)
msg_id: monotonically increasing integer
method: "plugin_notify"
params: [event_type, payload]
```

### Event Types

```lua
-- Needs input
{ "needs_input", {
    message = "Should I proceed?",
    options = { "Yes", "No", "Ask later" }
}}

-- Status update
{ "status", {
    level = "info",  -- "info", "warn", "error"
    message = "Running tests..."
}}

-- Complete
{ "complete", {
    summary = "Implementation done",
    files_changed = 5
}}

-- Error
{ "error", {
    message = "Failed to run tests",
    details = "..."
}}
```

## Functions

### `ipc.connect(socket_path)`
**Parameters:**
- `socket_path` (string, required)

**Returns:** rpc connection object or nil

### `ipc.notify(connection, event_type, payload)`
**Parameters:**
- `connection` (object, required)
- `event_type` (string, required)
- `payload` (table, required)

**Returns:** boolean success

### `ipc.disconnect(connection)`
Closes RPC connection.

## Neovim Handler

### RPC Server Setup
```lua
local function setup_ipc_handler()
  vim.fn.serverstart(function()
    -- Called when socket is created
    local socket_path = vim.v.servername
    
    -- Store for agents to connect
    save_socket_path(socket_path)
  end)
  
  -- Register RPC handler
  vim.rpcstart(vim.v.servername, {
    work_dispatch_notify = function(event, payload)
      handle_agent_notification(event, payload)
    end,
  })
end
```

### Notification Handler
```lua
local function handle_agent_notification(event, payload)
  if event == "needs_input" then
    -- Show notification
    require("notify")(payload.message, "info", {
      title = "Agent Input Required",
      actions = payload.options,
    })
    -- Update registry
    registry.update(payload.worktree_id, {
      needs_input = true,
      needs_input_since = os.date("!%Y-%m-%dT%H:%M:%SZ"),
    })
    -- Refresh picker if open
    picker.refresh()
    
  elseif event == "status" then
    -- Log status update
    log.info(payload.message)
    
  elseif event == "complete" then
    -- Notify completion
    require("notify")(payload.summary, "info", {
      title = "Agent Complete",
    })
  end
end
```

## Agent Side (Example)

Agents need to implement IPC client. For shell scripts:

```bash
#!/bin/bash
# Agent script sends notification via socat or similar

notify_needs_input() {
  local socket="$NVIM_LISTEN_ADDRESS"
  local msg='[2, 1, "plugin_notify", ["needs_input", {"message": "'"$1"'"}]]'
  echo -n "$msg" | socat - UNIX-CONNECT:"$socket"
}
```

## Acceptance Criteria

- [ ] Read NVIM_LISTEN_ADDRESS from environment
- [ ] Connect via msgpack-rpc socket
- [ ] Send notification: `{ method = "notify", params = ["event", payload] }`
- [ ] Graceful fallback if socket unavailable

## Error Handling

| Error | Behavior |
|-------|----------|
| Socket not found | Log warning, continue without IPC |
| Connection refused | Retry 3 times, then skip |
| Invalid message | Log error, ignore message |
| Write failure | Close connection, log error |
