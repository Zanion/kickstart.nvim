# SR-015: Reset Action

## Status
Draft

## Priority
Medium

## Dependencies
- SR-007: Agent Session Management
- SR-013: Merge Action (partially)

## Description
Reset an agent by re-launching it on the existing worktree.

## Constraints

### Reset Scope
- Resets agent session, not bead state (unless specified)
- Useful after reject or to restart a stuck agent
- Clears needs_input status

### Bead Handling
- Default: bead stays in current state (rejected)
- Optional: bead reset to "ready"

## Reset Process

```lua
local function reset(worktree_id, opts)
  opts = opts or {}
  
  local worktree = registry.get(worktree_id)
  if not worktree then
    return error("Worktree not found")
  end
  
  -- 1. Kill existing agent terminal if running
  if worktree.terminal_id then
    terminal.kill(worktree.terminal_id)
  end
  
  -- 2. Clear needs_input state
  needs_input.clear(worktree_id)
  
  -- 3. Update bead if requested
  if opts.reset_bead then
    beads.unclaim(worktree.bead_id)
    -- Or: beads.set_status(worktree.bead_id, "ready")
  end
  
  -- 4. Update registry
  registry.update(worktree_id, {
    status = "ready",
    needs_input = false,
    needs_input_since = nil,
    terminal_id = nil,
    terminal_buf = nil,
    reset_count = (worktree.reset_count or 0) + 1,
    last_reset = os.date("!%Y-%m-%dT%H:%M:%SZ"),
  })
  
  -- 5. Spawn new agent
  local session = agent.spawn(worktree, worktree.agent)
  
  -- 6. Update status to running
  registry.update(worktree_id, {
    session_id = session.id,
    terminal_id = session.terminal_id,
    status = "running",
  })
  
  -- 7. Claim bead if it was reset
  if opts.reset_bead then
    beads.claim(worktree.bead_id)
  end
  
  -- 8. Notify
  require("notify")(
    string.format("Agent reset for %s", worktree.bead_id),
    "info",
    { title = "Agent Reset" }
  )
  
  return { success = true, session = session }
end
```

## Functions

### `actions.reset(entry)`
Main reset action.

**Parameters:**
- `entry` (table, required) - Picker entry

**Returns:** boolean success

### `reset.confirm(worktree_id)`
Shows confirmation prompt.

```lua
function reset.confirm(worktree_id)
  local worktree = registry.get(worktree_id)
  
  local bead_will_reset = worktree.status == "rejected"
  
  local confirmed = vim.fn.confirm(
    string.format(
      "Reset agent for %s?\n\n".
      "Agent will be relaunched.\n".
      "%s",
      worktree.bead_id,
      bead_will_reset and "Bead will be reset to ready state." or ""
    ),
    { "Reset", "Cancel" },
    1
  )
  
  if confirmed == 1 then
    return reset(worktree_id, { reset_bead = bead_will_reset })
  end
end
```

## Availability

Reset is available for:
- `rejected` - Full reset, bead returns to ready
- `paused` - Resume, bead stays paused
- `done` - No reset needed

Reset is NOT available for:
- `running` - Already active
- `needs_input` - Clear needs_input instead

## State Transitions

### Rejected → Ready → Running
```
rejected (reset) → ready (spawn) → running
```

### Paused → Running
```
paused (focus) → running
```

## Validation

Before reset, check:
1. Worktree directory still exists
2. Git worktree is valid
3. No conflicting processes

```lua
function reset.validate(worktree_id)
  local worktree = registry.get(worktree_id)
  
  -- Check directory exists
  if not vim.fn.isdirectory(worktree.path) == 1 then
    return false, "Worktree directory not found"
  end
  
  -- Check git worktree is valid
  local result = vim.fn.system({
    "git", "worktree", "list", "--porcelain", worktree.path
  })
  if vim.v.shell_error ~= 0 then
    return false, "Git worktree invalid"
  end
  
  -- Check no locked files
  local locked = git.check_lock(worktree.path)
  if locked then
    return false, "Worktree has locked files"
  end
  
  return true
end
```

## Acceptance Criteria

- [ ] `<leader>R` triggers reset action (relaunches agent on same worktree)
- [ ] Bead status reset to previous state
- [ ] Agent terminal killed and respawned
- [ ] needs_input status cleared
- [ ] Session timestamp updated

## Edge Cases

| Case | Handling |
|------|----------|
| Worktree deleted | Show error, offer cleanup |
| Git conflicts | Show warning, offer to abort |
| Reset loop | Track reset count, warn after 3 |
| Agent crashes immediately | Don't infinite loop, mark failed |
