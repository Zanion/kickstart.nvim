# SR-004: Beads Integration Layer

## Status
Draft

## Priority
Critical

## Description
Integrate with `bd` CLI for beads task management.

## Dependencies
None (external CLI)

## Constraints

### CLI Flags
- Always use `--json` flag for machine-readable output
- Handle errors gracefully

### Parsing
- Parse output using `vim.json.decode()`
- Handle malformed JSON gracefully

### State Transitions
- Only allow valid state transitions per beads workflow

## Beads CLI Commands

### List Ready Beads
```bash
bd ready --json
```
**Output:**
```json
[
  {
    "id": "nvim-abc123",
    "title": "Add login functionality",
    "type": "feature",
    "priority": 1,
    "status": "ready"
  }
]
```

### Show Bead Details
```bash
bd show nvim-abc123 --json
```
**Output:**
```json
{
  "id": "nvim-abc123",
  "title": "Add login functionality",
  "description": "Implement user authentication...",
  "type": "feature",
  "priority": 1,
  "status": "ready",
  "created_at": "2024-01-01T00:00:00Z"
}
```

### Update Bead Status
```bash
bd update nvim-abc123 --claim --json
```

### Close Bead
```bash
bd close nvim-abc123 --reason "Implemented" --json
```

## Functions

### `beads.ready()`
- Fetches all beads with "ready" status
- Returns: table of bead objects

### `beads.show(id)`
- Fetches full details for a bead
- Returns: bead object or nil

### `beads.claim(id)`
- Claims a bead (sets to in_progress)
- Returns: boolean success

### `beads.unclaim(id)`
- Returns bead to ready state
- Returns: boolean success

### `beads.close(id, reason)`
- Closes bead with given reason
- Valid reasons: "Implemented", "Cancelled", "Duplicate"
- Returns: boolean success

### `beads.set_status(id, status)`
- Sets arbitrary status on bead
- Returns: boolean success

### `beads.parse_id(bead_id)`
- Extracts numeric/hash part from bead ID
- e.g., "nvim-abc123" → "abc123"
- Returns: string

## Acceptance Criteria

- [ ] Fetch ready beads: `bd ready --json`
- [ ] Show bead details: `bd show {id} --json`
- [ ] Update bead state: `bd update {id} --claim`
- [ ] Close bead on merge: `bd close {id} --reason "Implemented"`
- [ ] Parse bead metadata into structured format

## Error Handling

```lua
local function handle_beads_error(cmd, obj)
  if obj.code ~= 0 then
    local err = vim.json.decode(obj.stderr or "{}")
    return nil, {
      code = obj.code,
      message = err.message or "Beads command failed",
      cmd = cmd,
    }
  end
  return vim.json.decode(obj.stdout), nil
end
```
