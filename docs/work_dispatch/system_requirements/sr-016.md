# SR-016: Parallel Agent Support

## Status
Draft

## Priority
Medium

## Dependencies
- SR-002: Worktree Creation and Lifecycle
- SR-003: Worktree Registry
- SR-005: Dispatch Command

## Description
Support multiple agents working on the same bead in parallel.

## Constraints

### Parallel Limit
- Configurable max agents per bead
- Default: 5
- Prevents resource exhaustion

### Naming Collision
- Auto-increment counter in worktree name
- Unique branch names guaranteed

## Parallel Detection

When dispatching a bead:

```lua
local function dispatch.execute(bead_id, agent_name)
  -- Check existing worktrees for this bead
  local existing = registry.get_by_bead(bead_id)
  
  -- Get next counter
  local counter = worktree.get_counter(bead_id)
  
  -- Check limit
  local max_parallel = config.get("parallel.max_agents_per_bead", 5)
  if #existing >= max_parallel then
    return error(
      string.format(
        "Max parallel agents (%d) reached for %s",
        max_parallel,
        bead_id
      ),
      "LIMIT_EXCEEDED"
    )
  end
  
  -- Check if same agent already working
  for _, wt in ipairs(existing) do
    if wt.agent == agent_name and wt.status == "running" then
      -- Ask to use existing or create parallel
      local choice = vim.fn.confirm(
        string.format(
          "%s already working on %s.\n".
          "Use existing or create parallel?",
          agent_name,
          bead_id
        ),
        { "Use Existing", "Create Parallel", "Cancel" },
        1
      )
      
      if choice == 1 then
        return { existing = wt }
      elseif choice == 3 then
        return error("Cancelled", "CANCELLED")
      end
      -- choice == 2 continues to create parallel
    end
  end
  
  -- Continue with parallel creation
  ...
end
```

## Counter Management

```lua
local counters = {}

function worktree.get_counter(bead_id)
  if counters[bead_id] then
    counters[bead_id] = counters[bead_id] + 1
  else
    -- Initialize from existing worktrees
    local existing = registry.get_by_bead(bead_id)
    local max_counter = 0
    for _, wt in ipairs(existing) do
      local n = wt.name:match("%-(%d+)%-") or 0
      max_counter = math.max(max_counter, tonumber(n))
    end
    counters[bead_id] = max_counter
  end
  
  return counters[bead_id]
end
```

## Naming Algorithm

```lua
function worktree.generate_name(bead_id, bead_title)
  local slug = generate_slug(bead_title)
  local counter = worktree.get_counter(bead_id)
  
  if counter == 0 then
    -- First agent, no counter
    return string.format("feature-%s-%s", bead_id, slug)
  else
    -- Parallel agent, include counter
    return string.format("feature-%s-%d-%s", bead_id, counter, slug)
  end
end
```

## Independent Actions

Each parallel worktree is independent:
- Can merge one implementation
- Can reject others independently
- Each has own status, session, terminal

## Merge Scenario

User merges `claude` implementation:

1. Merge action on claude's worktree
2. claude's bead closed, PR created
3. Other worktrees marked as "superseded" or stay active
4. User can reject gemini/opencode
5. Or user can review all before acting

## Bead State

Bead state depends on all parallel worktrees:

| State | Meaning |
|-------|---------|
| ready | No agents working |
| in_progress | At least one agent running |
| done | First merge completed |
| (others) | Per-worktree status in registry |

## Functions

### `picker.show_parallel(bead_id)`
Shows only worktrees for a specific bead.

### `parallel.compare(worktree_ids)`
Opens comparison view (future enhancement).

## Acceptance Criteria

- [ ] Same bead can be dispatched to multiple agents
- [ ] Each agent gets a uniquely named worktree (e.g., `feature-bd-42-1`, `feature-bd-42-2`)
- [ ] Picker shows all parallel implementations
- [ ] User can merge one and reject others
- [ ] Counter increments correctly for same bead

## Configuration

```lua
require("work_dispatch").setup({
  parallel = {
    enabled = true,
    max_agents_per_bead = 5,
    allow_same_agent = true,  -- Same agent type twice
    auto_increment = true,
  }
})
```

## Edge Cases

| Case | Handling |
|------|----------|
| Max reached | Show error, list active agents |
| Counter collision | Regenerate with higher counter |
| Same agent type | Allow if config permits |
| All rejected | Bead stays in rejected state |
