# SR-001: Worktree Directory Management

## Status
Draft

## Priority
Critical

## Description
Manage the `.worktree/` directory structure for storing agent worktrees. The root location is configurable.

## Dependencies
None

## Constraints

### Path Configuration
- Worktree root defaults to `{repo_root}/.worktree/`
- Must support user-configurable path via setup
- Examples:
  - Relative: `.worktree/`
  - Absolute: `/home/user/agent-worktrees/`
  - Home-relative: `~/.agent-worktrees/`

### Git Ignore
- `.worktree/` directory must be gitignored
- Should add to `.gitignore` automatically if not present

## Configuration Schema

```lua
require("work_dispatch").setup({
  worktree_root = ".worktree/",  -- configurable
})
```

## Functions

### `worktree.ensure_root()`
- Creates worktree root directory if it doesn't exist
- Returns: absolute path to worktree root
- Errors: If parent directory doesn't exist

### `worktree.is_gitignored(root)`
- Checks if worktree root is in `.gitignore`
- Returns: boolean
- Side effect: Offers to add to `.gitignore` if missing

### `worktree.list()`
- Lists all worktrees in the root
- Returns: table of worktree info

### `worktree.resolve_path(path)`
- Resolves user-configured path to absolute
- Handles relative, absolute, and home-relative paths
- Returns: absolute path

## Acceptance Criteria

- [ ] Create `.worktree/` if it doesn't exist
- [ ] Respect configured worktree root path
- [ ] Ensure `.worktree/` is in `.gitignore`
- [ ] List all worktrees in the root

## Implementation Notes

```lua
-- Path resolution priority
1. Absolute path: use as-is
2. Home-relative (~): expand via os.expand()
3. Relative path: join with repo root via vim.fn.fnamemodify(get_repo_root(), ':p')
```
