# SR-002: Worktree Creation and Lifecycle

## Status
Draft

## Priority
Critical

## Description
Create and manage individual git worktrees for each agent task.

## Dependencies
- SR-001: Worktree Directory Management

## Constraints

### Git Worktree Commands
- Use `git worktree add` for creation
- Use `git worktree remove` for deletion
- Use `git worktree list` for listing

### Branch Naming
- Pattern: `feature-{bead-id}-{slug}`
- Parallel agents: `feature-{bead-id}-{n}-{slug}`
- Counter n starts at 1 for second agent

### Slug Generation
```lua
function generate_slug(title)
  return title
    :lower()
    :gsub("[^a-z0-9]+", "-")
    :gsub("^-+", "")
    :gsub("-+$", "")
    :sub(1, 50)
end
```

### Lifecycle States
- `creating` - Worktree being created
- `ready` - Worktree created, agent not yet spawned
- `running` - Agent active in worktree
- `paused` - Agent stopped, can be resumed
- `done` - Work merged
- `rejected` - Work rejected, kept for inspection

## Functions

### `worktree.create(opts)`
**Parameters:**
- `opts.bead_id` (string, required)
- `opts.agent` (string, required)
- `opts.slug` (string, optional, auto-generated)
- `opts.counter` (number, optional, auto-detected)

**Returns:**
```lua
{
  path = "/abs/path/to/worktree",
  branch = "feature-bd-42-add-login",
  name = "feature-bd-42-add-login",
  bead_id = "bd-42",
  agent = "gemini",
}
```

**Errors:**
- `EEXIST`: Worktree with same name exists
- `ENOTDIR`: Parent directory doesn't exist
- `EGIT`: Git command failed

### `worktree.remove(path)`
**Parameters:**
- `path` (string, required) - Absolute path to worktree

**Returns:** boolean success

### `worktree.exists(name)`
**Parameters:**
- `name` (string, required) - Worktree name

**Returns:** boolean

### `worktree.get_counter(bead_id)`
Returns the next counter value for parallel agents on same bead.

## Acceptance Criteria

- [ ] Create worktree with `git worktree add -b {branch} {path} {base}`
- [ ] Name format: `feature-{bead-id}-{slug}` or `feature-{bead-id}-{n}-{slug}`
- [ ] Track worktree metadata in registry
- [ ] Support cleanup command to remove worktrees manually

## Git Commands

```bash
# Create worktree
git worktree add -b feature-bd-42-add-login \
  .worktree/feature-bd-42-add-login \
  HEAD

# Remove worktree  
git worktree remove .worktree/feature-bd-42-add-login

# List worktrees
git worktree list --porcelain
```
