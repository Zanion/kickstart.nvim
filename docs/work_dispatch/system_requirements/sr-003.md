# SR-003: Worktree Registry

## Status
Draft

## Priority
Critical

## Description
Maintain a registry of active worktrees with metadata for tracking.

## Dependencies
- SR-002: Worktree Creation and Lifecycle

## Constraints

### Storage Location
- Registry stored at `{worktree_root}/registry.json`
- Human-readable for debugging

### Thread Safety
- Use vim.fn.system() for atomic file operations
- Or implement file locking

### Schema Version
- Include `version` field for migrations

## Registry Schema

```json
{
  "version": 1,
  "worktrees": [
    {
      "id": "uuid-v4",
      "path": "/abs/path/to/worktree",
      "branch": "feature-bd-42-add-login",
      "name": "feature-bd-42-add-login",
      "bead_id": "bd-42",
      "bead_title": "Add login functionality",
      "agent": "gemini",
      "session_id": "abc123",
      "status": "running",
      "needs_input": false,
      "needs_input_since": null,
      "created_at": "2024-01-15T10:30:00Z",
      "updated_at": "2024-01-15T11:45:00Z",
      "merge_commit": null,
      "pr_url": null
    }
  ]
}
```

### Status Values
| Status | Description |
|--------|-------------|
| `ready` | Worktree created, awaiting agent spawn |
| `running` | Agent actively working |
| `needs_input` | Agent awaiting user response |
| `paused` | Agent stopped by user |
| `done` | Work merged |
| `rejected` | Work rejected |

## Functions

### `registry.load()`
- Loads registry from disk
- Creates empty registry if doesn't exist
- Returns: registry table

### `registry.save(registry)`
- Writes registry to disk atomically
- Returns: boolean success

### `registry.add(worktree_info)`
- Adds new worktree to registry
- Updates `created_at` and `updated_at`
- Returns: new worktree entry

### `registry.update(id, updates)`
- Updates worktree entry by ID
- Updates `updated_at` timestamp
- Returns: updated entry or nil

### `registry.remove(id)`
- Removes worktree from registry
- Does NOT delete the worktree itself
- Returns: boolean success

### `registry.get(id)`
- Gets worktree by ID
- Returns: worktree entry or nil

### `registry.get_by_bead(bead_id)`
- Gets all worktrees for a bead
- Returns: table of worktree entries

### `registry.get_by_status(status)`
- Gets all worktrees with given status
- Returns: table of worktree entries

### `registry.list()`
- Lists all registered worktrees
- Returns: table of worktree entries

## Acceptance Criteria

- [ ] Store: worktree path, branch name, bead_id, agent type, session, status
- [ ] Read/write registry on worktree create/destroy
- [ ] Query registry for active agents
- [ ] Update agent status (running, needs_input, done, rejected)
